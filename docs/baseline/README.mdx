# NL Design System Baseline

<Paragraph lead>
  Wij maken technologie voor gebruikers. We leggen vast met welke browsers en andere hulpmiddelen deze technologie goed
  moet werken.
</Paragraph>

Het kernteam van NL Design system gebruikt deze baseline. Het is waar wij de Candidate en Hall of Fame componenten tegen testen.

De Baseline is niet verplicht voor Community bijdragen. We raden de Baseline wel aan. Organisaties kunnen de Baseline ook verplicht maken bij een opdracht of aanbesteding.

## Technologie met brede ondersteuning

Voor het bepalen of we een feature ondersteunen gebruiken we twee bronnen.

[Statistieken van RUM Archive ](https://rumarchive.com/insights/#ua-marketshare) geven aan welke browserversies veel gebruikt worden.

[Can I Use](https://www.caniuse.com/) geeft aan welke features ondersteund worden door de verschillende browsers en versies. We gebruiken features met het label "[Widely available across major browsers](https://web-platform-dx.github.io/web-features/#how-do-features-become-part-of-baseline%3F)".

Bijvoorbeeld: [CSS Grid Layout](https://caniuse.com/css-grid) wordt goed ondersteund en heeft het juiste label. Daarom gebruiken we `display: grid`. [CSS Subgrid](https://caniuse.com/css-subgrid) is eind 2024 niet goed ondersteund en heeft het label niet. Daarom gebruiken we CSS Subgrid nog niet.

## Software om mee te testen

Bij het testen gebruiken we de volgende combinaties van software samen met onze richtlijnen. Niet elke [screenreader](https://nldesignsystem.nl/woordenlijst/#screenreader) werkt optimaal met elke browser. De genoemde combinaties moeten geen problemen opleveren.

- Chrome met de screenreader [JAWS](https://www.freedomscientific.com/products/software/jaws/) op Windows
- Chrome met de screenreader [NVDA](https://www.nvaccess.org) op Windows
- Chrome op Android
- Chrome op Android met de screenreader [Talkback](https://support.google.com/accessibility/android/answer/6007100?hl%3Den)
- Chrome op Windows of macOS
- Firefox
- Firefox met de screenreader NVDA op Windows
- Safari op iOS
- Safari op iOS met de screenreader [VoiceOver](https://support.apple.com/en-gb/guide/iphone/iph3e2e415f/ios)
- Safari op macOS
- Safari op macOS met de screenreader [VoiceOver](https://support.apple.com/en-gb/guide/voiceover/welcome/mac)

Soms werkt er iets niet met oudere software. Dit kan ontdekt worden door testen maar ook door reacties van gebruikers. Overweeg dan wel het probleem op te lossen en te testen in specifieke oudere softwareversies.

## Testen voor toegankelijkheid

Een toegankelijkheidsonderzoek wordt vaak gedaan op basis van [WCAG-EM](https://www.w3.org/TR/WCAG-EM/). Bij deze aanpak ondersteunt men bepaalde technologie in de [Accessibility Support Baseline](https://www.w3.org/TR/WCAG-EM/#step1c).
Pagina's die NL Design System gebruiken moeten kunnen voldoen aan WCAG 2.2 Niveau AA. Dat is meer dan de wettelijk vereiste WCAG 2.1 AA.

Je kunt de NL Design System Baseline noemen bij je onderzoeksrapport onder "basisniveau van toegankelijkheidsondersteuning". Je kunt verwijzen naar de Baseline momentopname onder ["bij het onderzoek gebruikte browsers en softwareprogramma's"](https://www.digitoegankelijk.nl/toegankelijkheidsverklaring/controle).

## Momentopname

Technologie verandert steeds. Daarmee verandert de Baseline ook steeds. Op de NL Design System website delen we regelmatig een nieuwe Baseline. De laatste versie van de Baseline is de [Baseline september 2025](https://nldesignsystem.nl/baseline/2025-09/).

## Voor wie is de Baseline bedoeld?

De Baseline is in eerste instantie bedoeld voor ontwikkelingen waarin het kernteam een rol speelt:

- Candidate componenten
- Voorbeeld-webpagina's
- Toegankelijkheidsonderzoeken naar de NL Design System website en voorbeeld-webpagina's

## Hall of Fame

Voor Hall of Fame componenten vragen we nog feedback aan stakeholders zoals belangenorganisaties, testers en ontwikkelaars in de Community. Welke software en welke softwareversies zijn belangrijk om op te nemen in de Baseline?

[Deel je inzichten met het kernteam](/colofon), dat waarderen wij enorm!

### Community

De baseline is **niet verplicht** voor Community componenten en als je met NL Design System werkt.

Organisaties kunnen natuurlijk wel vrijwillig de NL Design System Baseline vereisen bij het aanbesteden van nieuwe ontwikkelingen en bij toegankelijkheidsonderzoeken, als ze een op zoek zijn naar een goede Accessibility Support Baseline.
