# Backup

<Paragraph lead>
  Wanneer je open source code aanbiedt op publieke infrastructuur, dan is het belangrijk dat je backups hebt waarvan de
  integriteit in orde is.
</Paragraph>

Gebruikers investeren tijd en energie in je open source project, en mensen kunnen afhankelijk zijn doorontwikkeling en veiligheidsupdates. Doe iedereen een plezier, en zorg dat er backups zijn die makkelijk teruggezet kunnen worden.

Er zijn goede manieren niet afhankelijk te zijn van een backup op je eigen computer. Lees meer hoe dit voor [github.com/nl-design-system](https://github.com/nl-design-system) werkt: [GitHub Backups](../../handboek/developer/05-infrastructuur/02-github.md#backup).
