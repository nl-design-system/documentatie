# Design Tokens lint

Gebruik de design-tokens-lint CLI tool om automatisch the tokens JSON bestanden te valideren.

## Installatie

```sh
pnpm add @nl-design-system-community/design-tokens-lint
```

## Gebruik

```shell
design-tokens-lint [opties] <bestand> [bestanden...]
```

### Valideer een tokens-bestand

```shell
design-tokens-lint src/tokens.json
```

### Valideer meerdere bestanden

Meerdere bestanden worden voor validatie in volgorde samengevoegd. Bij conflicten overschrijven latere bestanden eerdere.

```shell
design-tokens-lint base.json theme.json overrides.json
```

### Tokens Studio

Bij gebruik van een tokens.json die door Tokens Studio is gegenereerd moeten we 1 laag van de JSON afhalen.

```shell
design-tokens-lint --exclude-parent-keys figma/figma.tokens.json
```

### Uitvoer naar bestand schrijven

Schrijf uitvoer naar een bestand in plaats van stderr/stdout. Dit werkt voor zowel geslaagde als mislukte validaties. Bij succes wordt de volledige tokens-tree geschreven, inclusief aanvullende NL Design System-extensies en geüpgradede tokens die overeenkomen met [het Design Tokens JSON-formaat](https://www.designtokens.org/tr/2025.10/format/). Bij een fout worden de problemen als JSON naar het bestand geschreven.

```shell
design-tokens-lint --out result.json src/tokens.json
```

## Opties

| Optie                   | Beschrijving                                   |
| ----------------------- | ---------------------------------------------- |
| `--exclude-parent-keys` | Sluit bovenliggende sleutels uit van validatie |
| `--out`, `-o <bestand>` | Schrijf JSON-uitvoer naar bestand              |
| `--verbose`             | Toon stap-voor-stap voortgang                  |
| `--debug`               | Toon tussentijdse JSON-uitvoer                 |
| `--help`, `-h`          | Toon help                                      |
