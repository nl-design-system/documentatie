# Stappenplan

Aan de hand van dit stappenplan leggen we je graag uit hoe je gebruik kunt maken van de [NL Design System bibliotheek](figma-structuur.mdx) en de [Token Studio plugin](figma-structuur.mdx).

Heb je een vraag? Twijfel niet en [neem contact op met het kernteam](../../project/contact.mdx).

## Voordat je start

### Maak een Figma project aan

We adviseren je om binnen Figma een apart ‘Project’ voor je Design System aan te maken. Zo houd je een duidelijke scheiding tussen je Design System en overige design files.

[Meer informatie over Figma bestanden en projecten](https://help.figma.com/hc/en-us/sections/360006050633-Files-and-projects)

Meer informatie over Figma kun je vinden op [help.figma.com](https://help.figma.com/) en [forum.figma.com](https://forum.figma.com/).

### Maak een Github account aan

Om via de Token Studio plugin met hetzelfde JSON bestand te kunnen werken als developers heb je een Github account nodig.

[Maak een account aan bij github.com](https://github.com/)

## NL Design System bibliotheek

### Dupliceer de bibliotheek

[Video: Dupliceer de bibliotheek](https://youtu.be/BLzMVD6VmIw)

Doorloop de volgende stappen om de NL Design System bibliotheek te dupliceren.

1. Ga naar de Figma Community pagina van het NL Design System.
2. Klik op ‘NL DS - Bibliotheek (WIP)’.
3. Klik op ‘Get a copy’.
4. Hernoem het bestand naar: NL DS - [naam-van-je-organisatie] - Bibliotheek.
5. Verplaats het bestand naar het gewenste project.
6. Hernoem ‘Voorbeeld’ naar ‘[naam-van-je-organisatie]’ voor de cover van de Figma Bibliotheek. Je vindt de cover op de pagina ‘Helpers’.

![Screenshot van Figma. Een menu toont onder andere de optie 'Move to project'.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_library-move.jpg)

## Token Studio plugin

De plugin wordt regelmatig geupdate. Hierdoor kunnen onderstaande stappen (en screenshots) verouderd raken. Voor de meest up-to-date documentatie van de plugin ga je naar [docs.tokens.studio](https://docs.tokens.studio/).

### Activeer de plugin

[Video: Activeer de plugin](https://youtu.be/cY0yJ7KZOhg)

Doorloop de volgende stappen om de Token Studio plugin te activeren.

1. Open ‘NL DS - [naam-van-je-organisatie] - Bibliotheek’.
2. Klik op ‘Resources’.
3. Klik op ‘Plugins’.
4. Zoek op ‘Token Studio’.
5. Klik op ‘Run’ om de plugin te activeren.

![Screenshot van Figma. Een menu toon de plugin Token Studio.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-detail.jpg)

Er worden enkele onboarding schermen getoond. Die kun je doornemen of lekker skippen.

### Koppel de plugin aan Github

Eerlijk is eerlijk. Als je niet bekend bent met Github kunnen de volgende stappen wat lastig zijn. Loop je vast? Dan kun je wellicht een developer uit je team inschakelen. Of [neem contact op met het kernteam](../../project/contact.mdx).

[Video: Koppel de plugin aan Github](https://youtu.be/KTE_1ogE2T0)

Doorloop de volgende stappen om de Token Studio plugin aan Github te koppelen.

1. Zorg dat je de plugin open hebt staan.

![Screenshot toont de Studio Token plugin. Er zijn nog geen tokens zichtbaar.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-opened.jpg)

2. Klik op 'Settings'.

![Screenshot toont de Studio Token plugin. Het onderdeel 'Settings' is geselecteerd.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-settings.jpg)

3. Selecteer ‘Github’ bij Token Storage

![Screenshot toont de Studio Token plugin. Het onderdeel 'Settings' is geselecteerd. Binnen dit onderdeel is 'Github' geselecteerd.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-github.jpg)

4. Klik op ‘Add new credentials.

![Screenshot toont de Studio Token plugin. Het formulier 'Add new credentials' wordt getoond. Alle invulvelden zijn leeg.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-credentials-empty.jpg)

5. Vul de gegevens in:

**Name**

- Dit wordt enkel gebruikt voor de plugin zelf. Hier mag je van maken wat je wilt. Bijvoorbeeld; Tokens van [naam-van-je-organisatie].

**Personal Access Token**

- Een Personal Access Token heeft niks met Design Tokens te maken. De Personal Access token zorgt ervoor dat je jouw Github account kunt koppelen aan de plugin.
  - Heb je al een Personal Access Token? Vul deze in.
  - Heb je nog geen Personal Access Token? Volg dan de stappen onder [‘Een 'Personal Access Token' genereren’](#een-personal-access-token-genereren).

**Repository**

- Vul ‘nl-design-system/themes’ in.

**Default brach**

- Voeg je default brach toe. Dat is ‘main’.

**File Path**

- Ga naar https://github.com/nl-design-system/themes/tree/main/proprietary en bekijk of er voor jouw organisatie al tokens staan.
  - Is er nog geen thema voor jouw organisatie? [Neem contact op met het kernteam](../../project/contact.mdx).
- Voeg het pad in waar je tokens moeten worden opgeslagen. Waarschijnlijk is dat ‘proprietary/[naam-van-je-organisatie]-theme/figma/[naam-van-je-organisatie].tokens.json’

**baseUrl**

- Dit is optioneel. Hoef je niks mee te doen.

![Screenshot toont de Studio Token plugin. Het formulier 'Add new credentials' wordt getoond. Alle invulvelden zijn ingevuld.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-credentials-filled.jpg)

6. Klik op 'Save'.

De plugin zal aangeven dat er al tokens in de opgegeven ‘repo’ staan en vragen of je deze wilt binnen halen.

![Screenshot toont de Studio Token plugin. Er wordt een modal scherm getoond met de vraag 'Pull from Github?'. Opties zijn 'Cancel' en 'Yes'.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-pull.jpg)

7. Klik op ‘Yes’
8. Klik op ‘Tokens’
9. Voila, daar zul je de tokens hebben!

![Screenshot toont de Studio Token plugin. De tokensets Brand, Common en Componenten zijn zichtbaar. De checkboxes voor elke tokenset zijn niet geselecteerd.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-voila.jpg)

### Voer je huisstijl door

Nu is het zaak om de huisstijl van je organisatie door te voeren op de componenten van het NL Design System. Daarvoor is het handig dat je de [Design Token documentatie](../design-tokens/README.mdx) hebt gelezen.

Uiteraard is het ook slim om de documentatie van de Token Studio plugin bij de hand te houden. Deze vind je op [docs.tokens.studio](https://docs.tokens.studio/).

[Video: Voer je huisstijl door](https://youtu.be/-oiwSXPe6uI)

### Activeer de Brand, Common en Component token sets

Deze is makkelijk. Zet alle checkboxes aan. Door de checkbox van een token set aan te vinken wordt de token set actief. Zo kun je later bijvoorbeeld wisselen tussen een light of dark-mode.

![Screenshot toont de Studio Token plugin. De tokensets Brand, Common en Componenten zijn zichtbaar. De checkboxes voor elke tokenset zijn geselecteerd.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-checked.jpg)

### Hernoem ‘Voorbeeld’

Ga de tokens binnen de ‘brand’ token set af en hernoem ‘voorbeeld’ overal naar ‘[naam-van-je-organisatie]’.

![Screenshot toont de Studio Token plugin. Een context-menu toont onder andere de optie 'Rename'.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_rename.jpg)

Wanneer je gevraagd wordt hoe je wilt ‘remappen’ kies dan voor ‘Document’. Op die manier worden de wijzigingen direct over het gehele document doorgevoerd. Dit kan je later altijd weer aanpassen.

![Screenshot toont de Studio Token plugin. Er staat een cirkel om een button met de tekst 'Apply to document'.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_plugin-document.jpg)

### Voer Brand tokens door

Pak de huisstijl opties van je organisatie bij de hand. Voer de huisstijl opties van je organisatie door in de ‘brand’ token set.

Wijzig bijvoorbeeld de waarde van `[naam-van-je-organisatie].brand.primary.font-family` van `Source Serif Pro` naar `Comic Sans MS` of voeg een hele nieuwe kleuren set toe.

Bekijk tijdens het doorvoeren van de Brand tokens de betreffende Brand pagina binnen het Figma bestand. Zo krijg je direct visueel feedback op je keuzes.

**Tip!**
Laat de kleuren van het ‘Voorbeeld’ thema nog even staan tot je alle kleuren van jouw huisstijl aan Common en Component tokens hebt gerefereerd. Daarna kun je ze verwijderen.

**Let op!**
De tokens Font Family en Font Weight kunnen afzonderlijk van elkaar op tekst worden toegepast. Het is daarbij belangrijk om de Font Weight aan te houden zoals deze in Figma is aangegeven. Voor font family `Roboto` bijvoorbeeld `Bold` in plaats van iets als `700`.

![Screenshot van Figma en de Studio Token plugin. De pagina 'Typography' van het niveau 'Brand' is geselecteerd. Er is een deel van een Frame te zien met verschillende typografische onderdelen.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_adjust-brand.jpg)

### Voer Common tokens door

Voer de semantische keuzes voor je organisatie door in de ‘common’ token set.

Wijzig bijvoorbeeld de waarde van `nl.common.interaction.color` van `brand.violet.700` naar een blauwe tint uit de kleuren set die je zojuist in de Brand token set hebt toegevoegd.

Bekijk tijdens het doorvoeren van de Common tokens de betreffende Common én gerelateerde Component pagina’s binnen het Figma bestand. Zo krijg je direct visueel feedback op je keuzes.

De visuele voorbeelden, tokennamen en tokenwaardes worden gepresenteerd in begeleidende frames. Ze zijn gekoppeld aan de plugin dankzij de [‘Documentation token’ optie](https://docs.tokens.studio/tokens/documentation-tokens). Deze optie zorgt ervoor dat aanpassingen in naamgeving of waardes automatisch worden doorgevoerd in de begeleide frames.

![Screenshot van Figma en de Studio Token plugin. De pagina 'Color' van het niveau 'Common' is geselecteerd. Er is een deel van een Frame te zien met verschillende kleur onderdelen.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_adjust-common.jpg)

### Voer Component tokens door

Loop de componenten 1 voor 1 na in de ‘nl-components’ token set en pas waardes aan waar nodig.

Wijzig bijvoorbeeld de waarde van `nl.button.primary-action.background-color` van `nl.common.interaction.color` naar een groene tint uit de kleuren set die je zojuist in de Brand token set hebt toegevoegd.

![Screenshot van Figma en de Studio Token plugin. De pagina 'Button' van het niveau 'Component' is geselecteerd. Er is een deel van een Frame te zien met verschillende button design tokens en het button component zelf.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_adjust-component.jpg)

## Verstuur aanpassingen naar Github

[Video: Verstuur aanpassingen naar Github](https://youtu.be/5EgvruU2YyE)

Wanneer je de tokens naar wens hebt aangepast kun je deze wijzigingen in de bron doorvoeren. In Github termen noem je dat ‘Pushen’.

- Klik op ‘Push to Github’

![Screenshot toont de Studio Token plugin. Er staat een cirkel om een button met een omhoog wijzend pijl als icoon.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_github.jpg)

- Vul de gegevens in.
  - Commit message. Beschrijf hier kort wat je hebt aangepast.
  - Branch. ‘main’ staat al ingevuld. Het is echter verstandig om een aparte brach voor je wijzigingen aan te houden. Hoe je die branch noemt staat je vrij. Kies bijvoorbeeld voor ‘Brand update’ of ‘01-01-2023.
- Klik op ‘Push’.

![Screenshot toont de Studio Token plugin. Er wordt een modal scherm getoond verschillende invulvelden.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/meedoen_designers_stap_adjust-push.jpg)

De plugin stuurt de update naar Github en geeft je direct de mogelijkheid om een ‘Pull request’ te maken.

- Klik op ‘Create Pull Request’.

Vervolgens wordt je automatisch naar Github geleid waar je het Pull Request kunt maken. Bespreek met een developer uit je team hoe jullie verder met pull requests om gaan.

## Aanvullende informatie

### Een ‘Personal Access Token’ genereren

[Video: Personal Acces Token genereren](https://youtu.be/VTH_fWErFiQ)

**Ga direct naar [github.com/settings/tokens](github.com/settings/tokens) of volg de volgende stappen:**

- Klik op je avatar (rechtsboven)
- Klik op ‘Settings’
- Klik op het laatste menu item ‘Developer Settings’
- Klik op ‘Personal Access Token’
- Selecteer ‘Token (Classic)

**Genereer een token**

- Vul de gegevens in:
  - Note. Dit wordt enkel gebruikt voor github zelf. Hier mag je van maken wat je wilt. Bijvoorbeeld; Tokens van [naam-van-je-organisatie].
  - Expiration. Beslis zelf wanneer het token zou moeten verlopen.
  - Select scopes. Selecteer enkel ‘repo’.
- Klik op ‘Generate token’.
- Kopieer je Personal Access token.
- Vul de Personal Access token in bij de Github gegevens in de Token Studio plugin.

**Let op!**
Deel je token nooit met iemand die geen toegang zou moeten hebben tot je repository. Behandel het als je persoonlijke wachtwoord.
