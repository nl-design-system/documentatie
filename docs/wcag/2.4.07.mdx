<Summary />

Browsers tonen standaard een focusindicator. Dat is een blauwe, gele of zwarte omlijning, afhankelijk van de browser. Dit succescriterium vereist niet dat je een eigen indicator ontwerpt, maar wel dat je de bestaande niet verwijdert of onzichtbaar maakt. Als je de standaard vervangt door een eigen stijl, is die minstens even goed zichtbaar.

Een zichtbare focus kan er op verschillende manieren uitzien:

- Een omlijning rond het element.
- Een knipperend streepje in een invoerveld.
- Een achtergrondkleurverandering.
- Een dikke border onder een element.
- Vetgedrukte of onderstreepte tekst.

## Veelgemaakte fouten

### Focusindicator verwijderd in CSS

De standaard focusindicator van de browser wordt verwijderd met `outline: none`, zonder dat er een zichtbaar alternatief voor terugkomt. Toetsenbordgebruikers zien niet meer welk element de focus heeft en navigeren blind door de pagina.

**Hoe te testen:** zoek in de CSS naar `outline: none` en `outline: 0`. Controleer of er een `:focus-visible`-stijl is die een zichtbaar alternatief biedt.

<Guideline appearance="dont" title="Alle focus-outlines verwijderd">
    ```css
    *:focus {
      outline: none;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Outline alleen verwijderen met een zichtbaar alternatief via :focus-visible">
    ```css
    *:focus-visible {
      outline: 2px solid #0066cc;
      outline-offset: 2px;
    }
    ```
</Guideline>

Let op het verschil: `:focus-visible` toont de indicator alleen wanneer de browser detecteert dat je via het toetsenbord navigeert. Zo voorkom je de omlijning bij muisklikken maar behoud je hem voor toetsenbordgebruikers.

**Wie kan dit oplossen:** een developer vervangt `outline: none` door een `:focus-visible`-stijl.

### Focusindicator die opgaat in de achtergrond

De focusindicator is er, maar heeft dezelfde kleur als de achtergrond of is te subtiel om op te merken. Een lichtgrijze omlijning op een witte achtergrond is niet zichtbaar genoeg. Het contrast tussen de focusindicator en de achtergrond is minimaal 3:1.

**Hoe te testen:** tab naar een element en bekijk of de focusindicator opvalt. Test op alle achtergrondkleuren die op de website voorkomen. Gebruik een contrasttool om het contrast te meten.

<Guideline appearance="dont" title="Lichtgrijze outline op witte achtergrond">
    ```css
    button:focus-visible {
      outline: 1px solid #e0e0e0;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Contrastrijke outline">
    ```css
    button:focus-visible {
      outline: 2px solid #0066cc;
      outline-offset: 2px;
    }
    ```
</Guideline>

**Wie kan dit oplossen:** een developer past de kleur en dikte van de focusindicator aan.

### Achtergrondkleur of negatieve offset die de focusindicator bedekt

Een negatieve `outline-offset` duwt de focusindicator naar binnen, onder de achtergrondkleur van het element. Het element heeft visueel geen zichtbare indicator, terwijl die er technisch wel is.

**Hoe te testen:** tab naar elementen met een donkere achtergrond. Controleer in de DevTools of `outline-offset` op een negatieve waarde staat.

<Guideline appearance="dont" title="Negatieve offset duwt de outline onder het element">
    ```css
    a:focus-visible {
      outline: 2px solid black;
      outline-offset: -4px;
      background-color: black;
      color: white;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Positieve offset houdt de outline zichtbaar">
    ```css
    a:focus-visible {
      outline: 2px solid black;
      outline-offset: 2px;
    }
    ```
</Guideline>

**Wie kan dit oplossen:** een developer past de `outline-offset` aan naar een positieve waarde.

### Container met overflow hidden snijdt de focusindicator af

Een parent-element met `overflow: hidden` snijdt de focusindicator van een child-element af. De omlijning valt buiten de grenzen van de container en wordt niet getoond. Dit komt vaak voor bij kaartcomponenten met afgeronde hoeken.

**Hoe te testen:** tab naar interactieve elementen die in een container met afgeronde hoeken of een vaste hoogte staan. Controleer of de focusindicator volledig zichtbaar is.

<Guideline appearance="dont" title="Overflow hidden snijdt de focus-outline af">
    ```css
    .kaart {
      overflow: hidden;
      border-radius: 8px;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Voldoende padding voor de outline">
    ```css
    .kaart {
      overflow: hidden;
      border-radius: 8px;
      padding: 4px;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Outline vervangen door een inset box-shadow">
    ```css
    .kaart a:focus-visible {
      outline: none;
      box-shadow: inset 0 0 0 2px #0066cc;
    }
    ```
</Guideline>

**Wie kan dit oplossen:** een developer voegt padding toe aan de container of gebruikt een `box-shadow` in plaats van een `outline`.

### Focusindicator verwijderd via JavaScript

Een script roept `blur()` aan op elementen die focus ontvangen, of zet de `outline` dynamisch op `none`. Hierdoor kan de focus niet worden gevolgd met het toetsenbord.

**Hoe te testen:** zoek in de JavaScript-broncode naar `.blur()` en `outline = 'none'`.

<Guideline appearance="dont" title="Focus weghalen zodra een element het ontvangt">
    ```javascript
    element.addEventListener("focus", function () {
      this.blur();
    });
    ```
</Guideline>

<Guideline appearance="dont" title="Outline dynamisch verwijderen">
    ```javascript
    element.addEventListener("focus", function () {
      this.style.outline = "none";
    });
    ```
</Guideline>

Er is geen reden om focus programmatisch te verwijderen van een interactief element. Verwijder dit soort code.

**Wie kan dit oplossen:** een developer verwijdert de `blur()`-aanroepen en de dynamische `outline`-wijzigingen.

### Custom component zonder focusstijl

Bij het bouwen van eigen componenten zoals tabs, accordions en dropdowns wordt de focusstijl vergeten. De CSS van de component overschrijft de standaard browserindicator zonder dat er een alternatief voor terugkomt.

**Hoe te testen:** tab naar interactieve elementen in custom components. Controleer of elk element een zichtbare focusindicator heeft.

<Guideline appearance="dont" title="Component-CSS die de outline onbedoeld overschrijft">
    ```css
    .tab {
      border: none;
      outline: none;
      background: transparent;
      padding: 8px 16px;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Eigen focusstijl die past bij het ontwerp">
    ```css
    .tab {
      border: none;
      background: transparent;
      padding: 8px 16px;
    }

    .tab:focus-visible {
      outline: 2px solid #0066cc;
      outline-offset: 2px;
      border-radius: 4px;
    }
    ```

</Guideline>

**Wie kan dit oplossen:** een developer voegt een `:focus-visible`-stijl toe aan de component.

### Alleen kleurverandering als focusindicator

De focusindicator bestaat alleen uit een kleurverandering, zoals tekst die blauw wordt. Iemand die kleuren niet kan onderscheiden, ziet geen verschil. Combineer een kleurverandering altijd met een vormverandering zoals een omlijning, onderstreping of achtergrondmarkering.

**Hoe te testen:** bekijk de focusindicator in grijswaarden (gebruik de DevTools-optie "Emulate vision deficiencies" > "Achromatopsia"). Is het verschil tussen gefocust en niet-gefocust nog zichtbaar?

<Guideline appearance="dont" title="Alleen kleurverandering">
    ```css
    a:focus-visible {
      outline: none;
      color: blue;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Kleurverandering in combinatie met outline">
    ```css
    a:focus-visible {
      outline: 2px solid #0066cc;
      outline-offset: 2px;
      color: #0066cc;
    }
    ```
</Guideline>

**Wie kan dit oplossen:** een developer voegt een vormverandering toe aan de focusstijl.

## Hoe te testen

### Voor iedereen

1. Leg de muis aan de kant en navigeer met <kbd>Tab</kbd> door de pagina. Bij elk interactief element, zoals links, knoppen en formuliervelden, is duidelijk zichtbaar dat het de focus heeft.
2. Controleer of de focusindicator opvalt tegen de achtergrond. Test op pagina's met verschillende achtergrondkleuren.
3. Open dialoogvensters, dropdowns en andere interactieve onderdelen. Controleer of de focusindicator ook daar zichtbaar is.

### Voor developers

1. Zoek in de CSS naar `outline: none` en `outline: 0`. Controleer bij elke verwijzing of er een `:focus-visible`-stijl als alternatief is.
2. Controleer of `:focus-visible` consequent wordt gebruikt in plaats van `:focus` waar je de indicator alleen voor toetsenbordgebruikers wilt tonen.
3. Tab naar een element, open de <span lang="en">Styles</span>-tab in de DevTools en controleer welke focusstijlen actief zijn. Let op regels die de omlijning overschrijven.
4. Test elementen in containers met `overflow: hidden`. Tab naar elementen binnenin en controleer of de focusindicator niet wordt afgesneden.
5. Zoek in JavaScript naar `.blur()` en `outline = 'none'`.
6. Test in meerdere browsers: Chrome gebruikt een blauwe omlijning, Firefox een gestippelde lijn en Safari een blauwe gloed. Controleer of eigen stijlen in alle browsers werken.

## Gerelateerde succescriteria

- [2.1.1 Toetsenbord](https://nldesignsystem.nl/wcag/2.1.1/): alle interactieve elementen zijn bereikbaar met het toetsenbord. Een zichtbare focusindicator helpt alleen als elementen ook daadwerkelijk focus kunnen ontvangen.
- [2.4.3 Focusvolgorde](https://nldesignsystem.nl/wcag/2.4.3/): de volgorde waarin elementen focus krijgen is logisch. Lees hier meer als de focus in een onverwachte volgorde door de pagina springt.
- [2.4.11 Focusweergave](https://nldesignsystem.nl/wcag/2.4.11/): aanvullende eisen aan de focusindicator in WCAG 2.2, zoals minimale grootte en contrast. Lees hier meer als je een eigen focusindicator ontwerpt.
- [1.4.11 Contrast van niet-tekstuele content](https://nldesignsystem.nl/wcag/1.4.11/): de focusindicator is niet-tekstuele content en heeft een contrast van minimaal 3:1 nodig ten opzichte van de achtergrond.
