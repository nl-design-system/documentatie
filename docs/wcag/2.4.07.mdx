## Uitleg

<Summary />
Dit is essentieel voor iedereen die het toetsenbord gebruikt om te navigeren: mensen met een motorische beperking,
screenreadergebruikers in combinatie met een beeldscherm en mensen die de voorkeur geven aan toetsenbordnavigatie. Ook
voor mensen met cognitieve beperkingen is het belangrijk om te kunnen zien waar de focus zich bevindt.

Browsers geven standaard een focusindicator weer. Dat is vaak een blauwe, gele of zwarte omlijning. Dit succescriterium vereist niet dat je een eigen indicator ontwerpt, maar wel dat je de indicator niet verwijdert of onzichtbaar maakt. Als je de standaard vervangt door een eigen stijl, moet die minstens even goed zichtbaar zijn. In WCAG 2.2 is SC 2.4.11 bijgekomen, dat ook over focuszichtbaarheid gaat.

Wat telt als een zichtbare focus?

- Een knipperend verticaal streepje in een invoerveld.
- Een focusrand om een invoerveld. Het hoeft niet aan 4 kanten te zijn.
- De achtergrond van het element verandert.
- Een icoon is toegevoegd.
- Een dikke border onder een element.
- Tekst wordt vetgedrukt of onderstreept.

## Veelgemaakte fouten

Voor alle onderstaande voorbeelden geldt: als je `outline` een eigen kleur geeft, test deze kleur dan met alle kleuren die als achtergrond op je website worden gebruikt. Het contrast tussen de kleur van de <span lang="en">outline</span> en de achtergrond moet minimaal 3:1 zijn.

### Fout: <span lang="en">outline</span> verwijderen zonder alternatief

De meest voorkomende en schadelijkste fout. Designers vinden de standaard focus-<span lang="en">outline</span> lelijk en verwijderen die globaal met CSS. Toetsenbordgebruikers kunnen dan niet meer zien welk element de focus heeft.

```css
/* Fout: alle focus-outlines verwijderd */
*:focus {
  outline: none;
}

/* Goed: outline alleen verwijderen als je een duidelijk alternatief biedt */
*:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}
```

Let op het verschil: `:focus-visible` toont de indicator alleen wanneer de browser detecteert dat je via het toetsenbord navigeert. Zo voorkom je de <span lang="en">outline</span> bij muisklikken maar behoud je hem voor toetsenbordgebruikers.

### Fout: focusindicator die opgaat in de achtergrond

Een focusindicator die er technisch gezien is, maar dezelfde kleur heeft als de achtergrond of te subtiel is om op te merken, voldoet niet. De indicator moet zichtbaar genoeg zijn om op te vallen.

```css
/* Fout: lichtgrijze outline op witte achtergrond */
button:focus-visible {
  outline: 1px solid #e0e0e0;
}

/* Goed: contrastrijke outline */
button:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}
```

### Fout: achtergrondkleur of afbeelding die de <span lang="en">outline</span> bedekt

Als een element een achtergrondkleur heeft die even groot is als het element zelf, of als `outline-offset` op een negatieve waarde staat, kan de focusindicator verborgen raken achter het element of de achtergrond.

```css
/* Fout: negatieve offset duwt de outline onder het element */
a:focus-visible {
  outline: 2px solid black;
  outline-offset: -4px;
  background-color: black;
  color: white;
}

/* Goed: positieve offset houdt de outline zichtbaar */
a:focus-visible {
  outline: 2px solid black;
  outline-offset: 2px;
}
```

### Fout: <span lang="en">overflow hidden</span> op een parent-element dat de <span lang="en">outline</span> afsnijdt

Als een parent-element `overflow: hidden` heeft, kan de focusindicator van een child-element worden afgesneden en onzichtbaar worden. Dit is een subtiele fout die vaak over het hoofd wordt gezien.

```css
/* Fout: overflow hidden snijdt focus-outline af */
.kaart {
  overflow: hidden;
  border-radius: 8px;
}

/* Goed: overflow visible of voldoende padding */
.kaart {
  overflow: hidden;
  border-radius: 8px;
  padding: 4px; /* ruimte voor de outline */
}

/* Of: outline vervangen door een inset box-shadow */
.kaart a:focus-visible {
  outline: none;
  box-shadow: inset 0 0 0 2px #0066cc;
}
```

### Fout: focusindicator verwijderd via JavaScript

Scripts die focus actief verwijderen van elementen door `blur()` aan te roepen, of de <span lang="en">outline</span> dynamisch uitzetten, maken het onmogelijk om de focus te volgen.

```javascript
// Fout: focus weghalen zodra een element het ontvangt
element.addEventListener("focus", function () {
  this.blur();
});

// Fout: outline dynamisch verwijderen
element.addEventListener("focus", function () {
  this.style.outline = "none";
});
```

Er is geen reden om focus programmatisch te verwijderen van een interactief element. Verwijder dit soort code.

### Fout: <span lang="en">custom component</span> zonder focusstijl

Bij het bouwen van <span lang="en">custom components</span> zoals tabs, accordions en dropdowns wordt de focusstijl vaak vergeten. De standaard browser-<span lang="en">outline</span> wordt soms onbedoeld overschreven door de CSS van de component zonder dat er een alternatief voor terugkomt.

```css
/* Fout: component-CSS die de outline onbedoeld overschrijft */
.tab {
  border: none;
  outline: none; /* verwijderd om het ontwerp "schoon" te houden */
  background: transparent;
  padding: 8px 16px;
}

/* Goed: eigen focusstijl die past bij het ontwerp */
.tab {
  border: none;
  background: transparent;
  padding: 8px 16px;
}

.tab:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
  border-radius: 4px;
}
```

### Fout: alleen kleurverandering als focusindicator

Als de focusindicator uitsluitend bestaat uit een kleurverandering, is dat niet genoeg voor mensen die kleuren niet kunnen onderscheiden. Denk aan tekst die blauw wordt. Combineer kleurveranderingen altijd met een vormverandering zoals een <span lang="en">outline</span>, onderstreping of achtergrondmarkering.

```css
/* Fout: alleen kleurverandering */
a:focus-visible {
  outline: none;
  color: blue;
}

/* Goed: kleurverandering in combinatie met outline */
a:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
  color: #0066cc;
}
```

## Hoe te testen

### Voor iedereen

1. Leg je muis aan de kant en navigeer met <kbd>Tab</kbd> door de pagina.
2. Kun je bij elk interactief element, zoals links, knoppen en formuliervelden, duidelijk zien dat het de focus heeft?
3. Is de focusindicator goed zichtbaar? Valt hij op tegen de achtergrond?
4. Verdwijnt de focusindicator op geen enkel moment terwijl je door de pagina tabt?
5. Controleer ook elementen in modalen, dropdowns en andere interactieve onderdelen die pas na een actie zichtbaar worden.

### Voor developers

1. Zoek in je CSS naar `outline: none` en `outline: 0`. Elke keer dat je de <span lang="en">outline</span> verwijdert, moet er een zichtbaar alternatief zijn via `:focus-visible`.
2. Controleer of `:focus-visible` consequent wordt gebruikt in plaats van `:focus` waar je de indicator alleen voor toetsenbordgebruikers wilt tonen.
3. Inspecteer elementen met de browser DevTools: tab naar een element, open de <span lang="en">Styles</span>-tab en controleer welke focusstijlen actief zijn. Let op of er regels zijn die de <span lang="en">outline</span> overschrijven.
4. Test met `overflow: hidden` op containers. Tab naar elementen binnenin en controleer of de focusindicator niet wordt afgesneden.
5. Zoek in JavaScript naar `.blur()` en `outline = 'none'` om te controleren of focus niet programmatisch wordt verwijderd of verborgen.
6. Test in meerdere browsers: de standaard focusstijl verschilt per browser. Chrome gebruikt een blauwe <span lang="en">outline</span>, Firefox een gestippelde lijn en Safari een blauwe gloed. Zorg dat je <span lang="en">custom</span> stijlen in alle browsers werken.
