## Uitleg

<Summary />

Er is een belangrijk verschil tussen kleur en contrast. Als het verschil tussen twee kleuren een contrastverhouding heeft van minder dan 3:1, is er sprake van kleurgebruik. Bij een contrastverhouding van 3:1 of hoger spreekt WCAG van contrast. Contrast geldt als een andere manier om informatie over te brengen. Dan is dit succescriterium niet van toepassing.

## Veelgemaakte fouten

### Fout: links in lopende tekst zijn alleen door kleur te onderscheiden

Links die niet onderstreept zijn en alleen door kleur verschillen van de omringende tekst, zijn niet voor iedereen herkenbaar. Iemand met kleurenblindheid ziet geen verschil tussen de link en de rest van de tekst.

```css
/* Fout: links alleen door kleur te onderscheiden */
a {
  color: #2056a5;
  text-decoration: none;
}

/* Goed: onderstreping maakt links ook zonder kleur herkenbaar */
a {
  color: #2056a5;
  text-decoration: underline;
}

/* Of: combineer kleur met een andere visuele aanduiding */
a {
  color: #2056a5;
  text-decoration: none;
  font-weight: bold;
  border-bottom: 1px solid currentColor;
}
```

### Fout: formuliervelden kleuren rood bij een foutmelding zonder tekst

Als bij een fout in een invoerveld alleen de rand rood wordt, mist iemand met kleurenblindheid de foutmelding. Voeg altijd een tekstuele foutmelding toe die aangeeft wat er mis is.

```html
<!-- Fout: alleen een rode rand -->
<label for="email">E-mailadres</label>
<input id="email" type="email" style="border-color: red;" />

<!-- Goed: rode rand én een tekstuele foutmelding -->
<label for="email">E-mailadres</label>
<input id="email" type="email" aria-describedby="fout-email" aria-invalid="true" style="border-color: red;" />
<p id="fout-email" class="foutmelding">Vul een geldig e-mailadres in.</p>
```

### Fout: gekleurde achtergrond als enige manier om categorieën aan te geven

Cellen in een tabel of items in een lijst die alleen een andere achtergrondkleur hebben om ze in categorieën te verdelen, zijn niet voor iedereen te onderscheiden. Voeg een tekstlabel of icoon toe.

```html
<!-- Fout: alleen kleur geeft de status aan -->
<table>
  <tr>
    <td>Aanvraag 1</td>
    <td style="background-color: green;">Goedgekeurd</td>
  </tr>
  <tr>
    <td>Aanvraag 2</td>
    <td style="background-color: red;">Afgewezen</td>
  </tr>
</table>

<!-- Goed: kleur én tekst geven de status aan -->
<table>
  <tr>
    <td>Aanvraag 1</td>
    <td style="background-color: green; color: white;">✓ Goedgekeurd</td>
  </tr>
  <tr>
    <td>Aanvraag 2</td>
    <td style="background-color: red; color: white;">✗ Afgewezen</td>
  </tr>
</table>
```

### Fout: huidige pagina in het menu alleen in een andere kleur

De actieve pagina in een navigatiemenu die uitsluitend een andere tekstkleur heeft, is niet voor iedereen herkenbaar. Voeg een extra visuele aanduiding toe, zoals een onderstreping, een achtergrondkleur met voldoende contrast of een vetgedrukt lettertype. Een broodkruimelpad of een duidelijke paginatitel kan ook als alternatief dienen.

```css
/* Fout: alleen kleur geeft de actieve pagina aan */
.nav-link--actief {
  color: #0066cc;
}

/* Goed: kleur én een onderstreping */
.nav-link--actief {
  color: #0066cc;
  border-bottom: 3px solid currentColor;
  font-weight: bold;
}
```

### Fout: actieve knop in een carrousel alleen in kleur aangegeven

In een carrousel met ronde navigatieknoppen die de slides aangeven, mag de actieve knop niet alleen door kleur verschillen van de overige knoppen. Gebruik ook een andere vorm, een dikkere rand of een groter formaat.

```css
/* Fout: alleen kleur verschilt */
.dot {
  background-color: #ccc;
  border-radius: 50%;
  width: 12px;
  height: 12px;
}

.dot--actief {
  background-color: #0066cc;
}

/* Goed: kleur én formaat verschillen */
.dot--actief {
  background-color: #0066cc;
  width: 16px;
  height: 16px;
  border: 2px solid #003d7a;
}
```

### Fout: focusrand gebruikt alleen kleur

Als een zelfgeschreven focusrand minder dan 3:1 contrast heeft met de achtergrond, wordt alleen kleur gebruikt om de focuspositie aan te geven. Dat is onvoldoende voor bezoekers met een visuele beperking of kleurenblindheid.

```css
/* Fout: focusrand met onvoldoende contrast (alleen kleur) */
button:focus-visible {
  outline: 2px solid #99ccff; /* lichtblauw op wit: < 3:1 */
}

/* Goed: contrastrijke focusrand */
button:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}
```

### Fout: data in een grafiek alleen door kleur te onderscheiden

Lijnen of vlakken in een grafiek die alleen een andere kleur hebben, zijn voor mensen met kleurenblindheid niet te onderscheiden. Gebruik naast kleur ook een ander visueel kenmerk, zoals streepjes, stippellijnen of symbolen. Bied bij complexe grafieken ook een tabel aan met dezelfde data.

![Twee verschillende weergaven van een grafiek, een fout voorbeeld](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/wcag-1-4-1-grafiek-fout.png)

Gebruik naast kleur ook vorm om de data te onderscheiden, zoals stippeltjes of streepjes.

![Twee verschillende weergaven van een grafiek, een goed voorbeeld.](https://raw.githubusercontent.com/nl-design-system/documentatie/assets/wcag-1-4-1-grafiek-goed.png)

De data browsergebruik komt van [statcounter.com](https://gs.statcounter.com/browser-market-share#monthly-200901-202412).

### Fout: legenda bij een kaart gebruikt alleen kleur

In een legenda mag je niet uitsluitend kleur gebruiken om gebieden op een kaart te onderscheiden. Voeg naast kleur ook patronen, arceringen of labels toe. Een praktische controle: bekijk de kaart in zwart-wit. Is het onderscheid dan nog steeds duidelijk?

## Gerelateerde succescriteria

- [SC 1.3.1](https://nldesignsystem.nl/wcag/1.3.1/) — Info en relaties: informatie die visueel met kleur wordt gegeven, moet ook programmatisch beschikbaar zijn.
- [SC 1.3.3](https://nldesignsystem.nl/wcag/1.3.3/) — Zintuiglijke eigenschappen: verwijs niet alleen naar kleur in instructies.
- [SC 1.4.3](https://nldesignsystem.nl/wcag/1.4.3/) — Contrast (minimum): tekst moet voldoende contrasteren met de achtergrond.
- [SC 1.4.11](https://nldesignsystem.nl/wcag/1.4.11/) — Contrast van niet-tekstuele content.
- [SC 4.1.2](https://nldesignsystem.nl/wcag/4.1.2/) — Naam, rol, waarde: als de status van een interactief element met kleur wordt aangegeven, moet die informatie ook beschikbaar zijn voor screenreadergebruikers.

## Hoe te testen

### Voor iedereen

1. Bekijk de pagina en let op plekken waar kleur informatie overbrengt. Denk aan links, foutmeldingen, grafieken, statusindicatoren en navigatiemenu's.
2. Stel jezelf de vraag: als ik geen kleuren kon zien, zou ik dezelfde informatie dan nog steeds begrijpen?
3. Bekijk de pagina met een grijsfilter. Is alle informatie nog steeds te begrijpen?
4. Controleer of links in lopende tekst onderstreept of op een andere manier dan alleen kleur te onderscheiden zijn.
5. Controleer of foutmeldingen bij formuliervelden ook als tekst zichtbaar zijn.

### Voor developers

1. Gebruik een kleurenblindheidsimulator in je browser:
   - Firefox: open de inspector, kies de tab "Accessibility" en selecteer bij "Simulate" de optie "Achromatopsia".
   - Chrome: open de inspector, open de console via de drie puntjes linksboven, kies "Rendering" en selecteer onder "Emulate vision deficiencies" de optie "Achromatopsia".
2. Controleer links in lopende tekst. Zoek in je CSS naar links met `text-decoration: none` zonder een alternatieve visuele aanduiding.
3. Controleer formuliervalidatie. Wordt een fout alleen met een rode rand aangegeven? Zoek naar `border-color: red` of vergelijkbare stijlen zonder bijbehorende tekstuele foutmelding.
4. Controleer grafieken en diagrammen. Zijn de datareeksen alleen door kleur te onderscheiden, of ook door vorm, patroon of label?
5. Controleer navigatiemenu's. Is de actieve pagina alleen door kleur te herkennen?
6. Gebruik axe DevTools of WAVE voor een eerste scan. Sommige kleurproblemen worden automatisch gedetecteerd. De meeste vereisen handmatige controle.
