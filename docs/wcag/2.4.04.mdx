## Uitleg

<Summary />

De "context" die mag meetellen is beperkt tot wat programmatisch te bepalen is:

- Een zin of alinea waarin de link staat:
  `<p>Op onze <a href="#">website</a> lees je meer over ...</p>`
- Een lijstitem:
  `<ul><li>De tweede reden om <a href="#">ons boek</a> te lezen is ...</li></ul>`
- Een tabelcel (`<td>` of `<th>`):
  `<table><tr><td>Het leukste <a href="#">evenement</a> dit jaar was ...</td></tr></table>`

Informatie die alleen visueel duidelijk is (zoals de positie op de pagina of een afbeelding naast de link) telt niet mee. Wat valt **niet** onder linkcontext?

- Een kop waar de link onder staat
- Een alinea die boven een link staat
- Containers zoals `<article>`, `<section>`, `<div>`

Er is een uitzondering: als het doel van de link voor iedereen onduidelijk zou zijn, ook voor ziende bezoekers, dan is het geen schending van dit SC. Maar eerlijk: die uitzondering komt zelden voor.

## Veelgemaakte fouten

### Fout: "Lees meer" als linktekst

Als deze link onder een alinea of een kop van een bericht staat, heeft het geen context en is het linkdoel onduidelijk.
Dit is veruit de meest voorkomende fout. "Lees meer" zegt niets over de bestemming van de link. Iemand die met een screenreader door de links op een pagina navigeert, hoort tien keer "Lees meer" zonder te weten waar elke link naartoe gaat.

```html
<!-- Fout: geen context -->
<p>We hebben een nieuw retourbeleid.</p>
<a href="/retourbeleid">Lees meer</a>.

<!-- Goed: wel context -->
<p>We hebben een nieuw <a href="/retourbeleid">retourbeleid</a>. Lees de volledige voorwaarden.</p>
```

### Fout: link op een URL als zichtbare tekst

Een kale URL als linktekst is niet informatief en wordt door een screenreader karakter voor karakter of als een lange onbegrijpelijke reeks voorgelezen.

```html
<!-- Fout -->
<a href="https://www.example.com/rapport-2024.pdf">https://www.example.com/rapport-2024.pdf</a>

<!-- Goed -->
<a href="https://www.example.com/rapport-2024.pdf">Jaarrapport 2024 (PDF, 2,3 MB)</a>
```

### Fout: afbeelding als link zonder alt-tekst die het doel beschrijft

Als een link alleen een afbeelding bevat en die afbeelding geen alt-tekst heeft, dan heeft de link geen toegankelijke naam. Een screenreader leest dan de URL voor, of helemaal niets.

```html
<!-- Fout: geen alt -->
<a href="/winkelwagen">
  <img src="cart-icon.svg" />
</a>

<!-- Fout: alt beschrijft de afbeelding, niet het doel -->
<a href="/winkelwagen">
  <img src="cart-icon.svg" alt="Icoon van een winkelwagen" />
</a>

<!-- Goed: alt beschrijft het doel -->
<a href="/winkelwagen">
  <img src="cart-icon.svg" alt="Winkelwagen" />
</a>
```

### Fout: link opent in een nieuw venster

Als je ervoor kiest om deze informatie aan een ziende bezoeker te geven door het icoontje bij de link te plaatsen, dan moet je het ook aan hulpsoftware laten weten. Een link met `target="_blank"` opent een nieuwe browsertab en deze informatie kan belangrijk zijn voor je bezoekers.

```html
<!-- Fout -->
<a href="https://extern.nl/artikel" target="_blank"
  ><svg aria-hidden="true" class="icoon-extern"></svg>Lees het artikel</a
>

<!-- Goed -->
<a href="https://extern.nl/artikel" target="_blank"
  ><svg aria-hidden="true" class="icoon-extern"></svg>
  Lees het artikel (opent in nieuw venster)
</a>

<!-- Of met screenreadertekst -->
<a href="https://extern.nl/artikel" target="_blank">
  Lees het artikel
  <span class="sr-only">(opent in nieuw venster)</span>
  <svg aria-hidden="true" class="icoon-extern"><!-- icoon --></svg>
</a>
```

### Fout: hele alinea of groot blok tekst als link

Als je een heel tekstblok klikbaar maakt, wordt de volledige tekst als linktekst voorgelezen. Dat is onnodig lang en maakt het lastig om snel te begrijpen wat de link doet.

```html
<!-- Fout: hele alinea is een link -->
<a href="/product/jas">
  <p>
    Deze prachtige winterjas is gemaakt van gerecycled materiaal en houdt je warm tot -20 graden. Verkrijgbaar in drie
    kleuren. Nu met 20% korting.
  </p>
</a>

<!-- Goed: alleen de relevante tekst is een link -->
<p>
  Deze prachtige
  <a href="/product/jas">winterjas van gerecycled materiaal</a> houdt je warm tot -20 graden. Verkrijgbaar in drie
  kleuren. Nu met 20% korting.
</p>
```

### Fout: link zonder enige tekst of toegankelijke naam

Een lege link — bijvoorbeeld een `<a>`-element met alleen een achtergrondafbeelding via CSS of een leeg element — heeft geen toegankelijke naam. Een screenreader meldt een link, maar niet waarheen.

```html
<!-- Fout: lege link met alleen een CSS-achtergrond -->
<a href="/home" class="logo-link"></a>

<!-- Goed -->
<a href="/home" class="logo-link">
  <span class="sr-only">Acme company</span>
</a>
```

### Fout: linktekst is verborgen voor hulpsoftware

Als je linktekst verbergt met `display: none` of `visibility: hidden`, dan is die tekst ook onzichtbaar voor screenreaders. De link heeft dan geen inhoud en dus geen linkdoel.

```html
<!-- Fout: tekst verborgen met display:none -->
<a href="/acties">
  <span style="display: none;">Bekijk onze acties</span>
</a>

<!-- Goed: visueel verborgen maar wel beschikbaar voor screenreaders -->
<a href="/acties">
  <span class="sr-only">Bekijk onze acties</span>
</a>
```

### Fout: linktekst in een paginering is niet beschrijvend genoeg

Als je paginering alleen uit cijfers bestaat zonder verdere context, is het linkdoel onduidelijk. "1", "2", "3" zegt niets als je niet weet dat het om pagina's gaat. Dit geldt ook voor carrousels met genummerde navigatie.

```html
<!-- Fout: alleen cijfers zonder context -->
<ul>
  <li><a href="?pagina=1">1</a></li>
  <li><a href="?pagina=2">2</a></li>
  <li><a href="?pagina=3">3</a></li>
</ul>

<!-- Goed: als navigatie met aria-labels -->
<nav aria-label="Paginering">
  <ul>
    <li><a href="?pagina=1" aria-label="Pagina 1">1</a></li>
    <li><a href="?pagina=2" aria-label="Pagina 2">2</a></li>
    <li><a href="?pagina=3" aria-label="Pagina 3">3</a></li>
  </ul>
</nav>
```

### Fout: linktekst bestaat uit een enkel teken

Een losse letter of icoon als linktekst maakt het linkdoel niet duidelijk. Dit zie je bijvoorbeeld bij knoppen met A, A, A om de tekstgrootte aan te passen, of bij social media-iconen zonder label.

```html
<!-- Fout: enkele letters zonder context -->
<a href="?size=small">A</a>
<a href="?size=medium">A</a>
<a href="?size=large">A</a>

<!-- Goed: met duidelijke labels -->
<a href="?size=small" aria-label="Kleine tekst">A</a>
<a href="?size=medium" aria-label="Normale tekst">A</a>
<a href="?size=large" aria-label="Grote tekst">A</a>
```

### Fout: linktekst bevat dubbele informatie

Als een link zowel een afbeelding met alt-tekst als zichtbare tekst bevat, worden die samengevoegd tot de linktekst. Dat levert dubbelingen op: een screenreader leest dan iets als "Winkelwagen Winkelwagen" of "Zoek zoek".

```html
<!-- Fout: alt-tekst en zichtbare tekst zeggen hetzelfde -->
<a href="/winkelwagen">
  <img src="cart.svg" alt="Winkelwagen" />
  Winkelwagen
</a>
<!-- Screenreader leest: "Winkelwagen Winkelwagen" -->

<!-- Goed: lege alt zodat alleen de zichtbare tekst meetelt -->
<a href="/winkelwagen">
  <img src="cart.svg" alt="" />
  Winkelwagen
</a>
```

### Fout: linktekst bevat tekens die er niet in horen

Tekens als `>` en `>>` worden door screenreaders voorgelezen als "groter dan". "Lees meer >>" wordt dus "Lees meer groter dan groter dan". Pijltjes en andere decoratieve tekens horen niet in je linktekst.

```html
<!-- Fout -->
<a href="/aanbiedingen">Bekijk aanbiedingen >></a>
<!-- Screenreader: "Bekijk aanbiedingen groter dan groter dan" -->

<!-- Goed -->
<a href="/aanbiedingen">Bekijk aanbiedingen</a>

<!-- Of met een visueel pijltje dat verborgen is voor screenreaders -->
<a href="/aanbiedingen">
  Bekijk aanbiedingen
  <span aria-hidden="true"> &rarr;</span>
</a>
```

### Fout: linkcontext is alleen visueel te herkennen

Als een link met "Download" in een tabel staat maar die tabel mist de juiste HTML-markering, dan is de relatie tussen de link en de context (wat wordt hier gedownload?) niet programmatisch te bepalen.

```html
<!-- Fout: geen echte tabelmarkering, context is alleen visueel -->
<div class="rij">
  <div class="cel">Jaarverslag 2024</div>
  <div class="cel"><a href="/jaarverslag.pdf">Download</a></div>
</div>

<!-- Goed: echte tabel, context is programmatisch te bepalen -->
<table>
  <tr>
    <td>Jaarverslag 2024</td>
    <td><a href="/jaarverslag.pdf">Download</a></td>
  </tr>
</table>
```

### Fout: linktekst op een Nederlandstalige website is in het Engels

Als een `alt`-tekst of `aria-label` de linktekst bepaalt en die is in het Engels op een Nederlandstalige pagina, dan leest een screenreader die tekst voor met Nederlandse uitspraak. Dat maakt de tekst onverstaanbaar. Zorg dat de taal van de linktekst overeenkomt met de paginataal, of geef het element een `lang`-attribuut.

```html
<!-- Fout: Engels aria-label op een NL-pagina, wordt met NL-uitspraak voorgelezen -->
<a href="/contact" aria-label="Get in touch">
  <img src="contact.svg" alt="" />
</a>

<!-- Goed: Nederlands label -->
<a href="/contact" aria-label="Neem contact op">
  <img src="contact.svg" alt="" />
</a>

<!-- Of: met taalattribuut als het Engels moet blijven -->
<a href="/contact" aria-label="Get in touch" lang="en">
  <img src="contact.svg" alt="" />
</a>
```

### Advies: link die naar een bestand verwijst zonder het bestandstype te noemen

Links naar PDF's, Word-documenten, spreadsheets of andere bestanden moeten het bestandstype vermelden. Je bezoekers verwachten een webpagina. Een onverwachte download of een PDF die ineens in de browser opent, is verwarrend.

```html
<!-- Fout -->
<a href="/voorwaarden.pdf">Algemene voorwaarden</a>

<!-- Goed -->
<a href="/voorwaarden.pdf">Algemene voorwaarden (PDF, 340 KB)</a>
```

## Hoe te testen

### Visuele inspectie

1. Lees alle links op de pagina en vraag jezelf af: begrijp ik waar deze link naartoe gaat? Antwoord is Ja is goed.
2. Nee: korte of onduidelijke linktekst. Staat de link binnen context die het doel duidelijk maakt? Antwoord is Ja is goed. Maar controleer nog even de HTML.
3. Alle andere situaties, behalve navigaties, moet je verder in de code onderzoeken.

### Inspecteer de HTML

1. Open de Accessibility-tab in DevTools en inspecteer de toegankelijke naam van elke link. Controleer of die het doel beschrijft.
2. Of gebruik axe DevTools of WAVE om links zonder toegankelijke naam automatisch te vinden. Inhoudelijke kwaliteit van linktekst moet je handmatig controleren.
