<Summary />

De "context" die mag meetellen is beperkt tot elementen die in code een directe relatie hebben met de link:

- Een zin of alinea waarin de link staat:
  `<p>Op onze <a href="#">website</a> lees je meer over ...</p>`
- Een lijstitem:
  `<ul><li>De tweede reden om <a href="#">ons boek</a> te lezen is ...</li></ul>`
- Een tabelcel (`<td>` of `<th>`):
  `<table><tr><td>Het leukste <a href="#">evenement</a> dit jaar was ...</td></tr></table>`

Relaties tussen elementen die alleen visueel duidelijk zijn, tellen niet mee. Denk aan de positie op de pagina of een afbeelding naast de link. Dit valt niet onder linkcontext:

- Een kop waar de link onder staat.
- Een alinea die boven een link staat.
- Containers zoals `<article>`, `<section>`, `<div>`.

Er is een uitzondering: als het doel van de link voor iedereen onduidelijk zou zijn, ook voor ziende bezoekers, dan is het geen fout onder dit succescriterium.

## Veelgemaakte fouten

### "Lees meer" als linktekst

Dit is de meest voorkomende fout. "Lees meer" zegt niets over de bestemming van de link. Iemand die met een screenreader door de links op een pagina navigeert, hoort tien keer "Lees meer" zonder te weten waar elke link naartoe gaat.

**Hoe te testen:** open de linkslijst in een screenreader en controleer of elke linktekst op zichzelf begrijpelijk is.

<Guideline appearance="dont" title="'Lees meer' als linktekst zonder context.">

```html
<p>We hebben een nieuw retourbeleid.</p>
<a href="/retourbeleid">Lees meer</a>.
```

</Guideline>

<Guideline appearance="do" title="Linktekst met context in dezelfde zin.">

```html
<p><a href="/retourbeleid">Lees meer</a> over de retourvoorwaarden.</p>
```

</Guideline>

**Wie kan dit oplossen:** een contentmaker schrijft een beschrijvende linktekst. Een developer plaatst de link in de juiste context.

### Afbeelding als link zonder alt-tekst die het doel beschrijft

Als een link alleen een afbeelding bevat en die afbeelding geen alt-tekst heeft, dan heeft de link geen toegankelijke naam. Een screenreader leest dan de URL voor, of helemaal niets.

**Hoe te testen:** inspecteer de link in de Accessibility Tree. Heeft de link een toegankelijke naam die het doel beschrijft?

<Guideline appearance="dont" title="Afbeelding als link zonder alt-tekst.">

```html
<a href="/winkelwagen">
  <img src="cart-icon.svg" />
</a>
```

</Guideline>

<Guideline appearance="dont" title="Alt-tekst beschrijft de afbeelding in plaats van het linkdoel.">

```html
<a href="/winkelwagen">
  <img src="cart-icon.svg" alt="Icoon van een winkelwagen" />
</a>
```

</Guideline>

<Guideline appearance="do" title="Alt-tekst beschrijft het linkdoel.">

```html
<a href="/winkelwagen">
  <img src="cart-icon.svg" alt="Winkelwagen" />
</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer voegt een alt-tekst toe die het linkdoel beschrijft.

### Link opent in een nieuw venster zonder indicatie

Als een link in een nieuw venster opent (`target="_blank"`), moeten alle bezoekers dat weten. Een icoontje alleen is niet voldoende: geef de informatie ook door aan hulpsoftware.

**Hoe te testen:** zoek in de code naar `target="_blank"`. Staat er in de linktekst of via visueel verborgen tekst dat de link in een nieuw venster opent?

<Guideline appearance="dont" title="Link opent in nieuw venster zonder indicatie.">

```html
<a href="https://extern.nl/artikel" target="_blank">
  <svg aria-hidden="true" class="icoon-extern"></svg>
  Lees het artikel
</a>
```

</Guideline>

<Guideline appearance="do" title="Informatie over nieuw venster in de linktekst.">

```html
<a href="https://extern.nl/artikel" target="_blank">
  <svg aria-hidden="true" class="icoon-extern"></svg>
  Lees het artikel (opent in nieuw venster)
</a>

<!-- Of met visueel verborgen tekst: -->
<a href="https://extern.nl/artikel" target="_blank">
  Lees het artikel
  <span class="sr-only">(opent in nieuw venster)</span>
  <svg aria-hidden="true" class="icoon-extern"></svg>
</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer voegt "(opent in nieuw venster)" toe aan de linktekst of als visueel verborgen tekst.

### Hele alinea of groot blok als link

Als je een heel tekstblok klikbaar maakt, leest de screenreader de volledige tekst als linktekst voor. Dat is onnodig lang en maakt het lastig om snel te begrijpen wat de link doet.

**Hoe te testen:** inspecteer de toegankelijke naam van de link. Is die onnodig lang?

<Guideline appearance="dont" title="Hele alinea als link.">

```html
<a href="/product/jas">
  <p>
    Deze prachtige winterjas is gemaakt van gerecycled materiaal en houdt je warm tot -20 graden. Verkrijgbaar in drie
    kleuren. Nu met 20% korting.
  </p>
</a>
```

</Guideline>

<Guideline appearance="do" title="Alleen de relevante tekst als link.">

```html
<p>
  Deze prachtige <a href="/product/jas">winterjas van gerecycled materiaal</a> houdt je warm tot -20 graden.
  Verkrijgbaar in drie kleuren. Nu met 20% korting.
</p>
```

</Guideline>

**Wie kan dit oplossen:** een developer beperkt de link tot de relevante tekst.

### Link zonder inhoud of toegankelijke naam

Een lege link heeft geen toegankelijke naam. Denk aan een `<a>`-element met alleen een achtergrondafbeelding via CSS of een leeg element. Een screenreader meldt een link, maar niet waarheen.

**Hoe te testen:** gebruik axe DevTools of WAVE om links zonder toegankelijke naam automatisch te vinden.

<Guideline appearance="dont" title="Lege link zonder toegankelijke naam.">

```html
<a href="/home" class="logo-link"></a>
```

</Guideline>

<Guideline appearance="do" title="Link met visueel verborgen tekst voor screenreaders.">

```html
<a href="/home" class="logo-link">
  <span class="sr-only">Acme company</span>
</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer voegt visueel verborgen tekst of een `aria-label` toe.

### Linktekst verborgen voor hulpsoftware

Als je linktekst verbergt met `display: none` of `visibility: hidden`, dan is die tekst ook onzichtbaar voor screenreaders. De link heeft dan geen inhoud en dus geen linkdoel.

**Hoe te testen:** inspecteer de link in de Accessibility Tree. Heeft de link een toegankelijke naam?

<Guideline appearance="dont" title="Linktekst verborgen met display:none.">

```html
<a href="/acties">
  <span style="display: none">Bekijk onze acties</span>
</a>
```

</Guideline>

<Guideline appearance="do" title="Visueel verborgen maar beschikbaar voor screenreaders.">

```html
<a href="/acties">
  <span class="sr-only">Bekijk onze acties</span>
</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer gebruikt een `sr-only`-class in plaats van `display: none`.

### Paginering zonder beschrijvende linktekst

Als paginering alleen uit cijfers bestaat zonder verdere context, is het linkdoel onduidelijk. "1", "2", "3" zegt niets als je niet weet dat het om pagina's gaat.

**Hoe te testen:** open de linkslijst in een screenreader. Zijn de pagineringlinks begrijpelijk?

<Guideline appearance="dont" title="Paginering met alleen cijfers zonder context.">

```html
<ul>
  <li>
    <a href="?pagina=1">1</a>
  </li>
  <li>
    <a href="?pagina=2">2</a>
  </li>
  <li>
    <a href="?pagina=3">3</a>
  </li>
</ul>
```

</Guideline>

<Guideline appearance="do" title="Paginering met aria-labels.">

```html
<nav aria-label="Paginering">
  <ul>
    <li>
      <a href="?pagina=1" aria-label="Pagina 1">1</a>
    </li>
    <li>
      <a href="?pagina=2" aria-label="Pagina 2">2</a>
    </li>
    <li>
      <a href="?pagina=3" aria-label="Pagina 3">3</a>
    </li>
  </ul>
</nav>
```

</Guideline>

**Wie kan dit oplossen:** een developer voegt `aria-label` toe aan de pagineringlinks.

### Linktekst bestaat uit een enkel teken

Een losse letter of icoon als linktekst maakt het linkdoel niet duidelijk. Dit zie je bijvoorbeeld bij knoppen met A, A, A om de tekstgrootte aan te passen, of bij social media-iconen.

**Hoe te testen:** controleer of links met een enkel teken een `aria-label` hebben dat het doel beschrijft.

<Guideline appearance="dont" title="Enkele letters als linktekst zonder context.">

```html
<a href="?size=small">A</a>
<a href="?size=medium">A</a>
<a href="?size=large">A</a>
```

</Guideline>

<Guideline appearance="do" title="Letters met duidelijke aria-labels.">

```html
<a href="?size=small" aria-label="Kleine tekstgrootte">A</a>
<a href="?size=medium" aria-label="Normale tekstgrootte">A</a>
<a href="?size=large" aria-label="Grote tekstgrootte">A</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer voegt een `aria-label` toe dat het doel beschrijft.

### Linktekst bevat dubbele informatie

Als een link zowel een afbeelding met alt-tekst als zichtbare tekst bevat, voegt de screenreader die samen tot de linktekst. Dat levert dubbelingen op: een screenreader leest dan "Winkelwagen Winkelwagen".

**Hoe te testen:** inspecteer de toegankelijke naam van de link in de Accessibility Tree. Wordt informatie herhaald?

<Guideline appearance="dont" title="Alt-tekst en zichtbare tekst zeggen hetzelfde.">

```html
<a href="/winkelwagen">
  <img src="cart.svg" alt="Winkelwagen" />
  Winkelwagen
</a>
```

</Guideline>

<Guideline appearance="do" title="Lege alt zodat alleen de zichtbare tekst meetelt.">

```html
<a href="/winkelwagen">
  <img src="cart.svg" alt="" />
  Winkelwagen
</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer maakt de alt-tekst leeg (`alt=""`) als er al zichtbare linktekst is.

### Linktekst bevat decoratieve tekens

Tekens als `>` en `>>` leest een screenreader voor als "groter dan". "Lees meer >>" wordt "Lees meer groter dan groter dan". Decoratieve tekens horen niet in de linktekst.

**Hoe te testen:** lees de linktekst voor met een screenreader. Worden er onbedoelde tekens voorgelezen?

<Guideline appearance="dont" title="Decoratieve tekens in de linktekst.">

```html
<a href="/aanbiedingen">Bekijk aanbiedingen &gt;&gt;</a>
```

</Guideline>

<Guideline appearance="do" title="Decoratief pijltje verborgen voor screenreaders.">

```html
<a href="/aanbiedingen">Bekijk aanbiedingen</a>

<!-- Of met een decoratief pijltje: -->
<a href="/aanbiedingen">
  Bekijk aanbiedingen
  <span aria-hidden="true"> &rarr;</span>
</a>
```

</Guideline>

**Wie kan dit oplossen:** een developer verbergt decoratieve tekens met `aria-hidden="true"`.

### Linkcontext is alleen visueel te herkennen

Als een link met "Download" in een tabel staat maar die tabel mist de juiste HTML-markering, dan is de relatie tussen de link en de context niet programmatisch te bepalen.

**Hoe te testen:** controleer of de context van de link beschikbaar is in de Accessibility Tree.

<Guideline appearance="dont" title="Linkcontext alleen visueel herkenbaar door div-opmaak.">

```html
<div class="rij">
  <div class="cel">Jaarverslag 2024</div>
  <div class="cel">
    <a href="/jaarverslag.pdf">Download</a>
  </div>
</div>
```

</Guideline>

<Guideline appearance="do" title="Linkcontext programmatisch te bepalen via echte tabel.">

```html
<table>
  <tbody>
    <tr>
      <th>Jaarverslag 2024</th>
      <td>
        <a href="/jaarverslag.pdf">Download</a>
      </td>
    </tr>
  </tbody>
</table>
```

</Guideline>

**Wie kan dit oplossen:** een developer gebruikt de juiste HTML-elementen zodat de context programmatisch bepaalbaar is.

### Link naar een bestand zonder het bestandstype te noemen

Links naar PDF's, Word-documenten, spreadsheets of andere bestanden moeten het bestandstype vermelden. Je bezoekers verwachten een webpagina. Een onverwachte download of een PDF die in de browser opent, is verwarrend.

**Hoe te testen:** controleer of links naar bestanden het bestandstype en bij voorkeur de bestandsgrootte vermelden.

<Guideline appearance="dont" title="Link naar bestand zonder bestandstype.">

```html
<a href="/voorwaarden.pdf">Algemene voorwaarden</a>
```

</Guideline>

<Guideline appearance="do" title="Bestandstype en -grootte vermeld in de linktekst.">

```html
<a href="/voorwaarden.pdf">Algemene voorwaarden (PDF, 340 KB)</a>
```

</Guideline>

**Wie kan dit oplossen:** een contentmaker of developer voegt het bestandstype en de grootte toe aan de linktekst.

### URL als zichtbare linktekst

Een kale URL als linktekst is niet informatief. Een screenreader leest de URL karakter voor karakter of als een lange onbegrijpelijke reeks voor.

**Hoe te testen:** zoek op de pagina naar links waarvan de zichtbare tekst een URL is.

<Guideline appearance="dont" title="URL als zichtbare linktekst.">

```html
<a href="https://www.example.com/rapport-2024.pdf">https://www.example.com/rapport-2024.pdf</a>
```

</Guideline>

<Guideline appearance="do" title="Beschrijvende linktekst.">

```html
<a href="https://www.example.com/rapport-2024.pdf">Jaarrapport 2024 (PDF, 2,3 MB)</a>
```

</Guideline>

**Wie kan dit oplossen:** een contentmaker vervangt de URL door een beschrijvende linktekst.

## Hoe te testen

### Voor iedereen

1. Lees alle links op de pagina. Begrijp je waar elke link naartoe gaat, zonder de rest van de tekst te lezen?
2. Let op links als "Klik hier", "Lees meer", "Meer info", "Hier" en "Dit". Als deze vaak voorkomen, is er waarschijnlijk een probleem.
3. Controleer of links naar bestanden (PDF, Word) het bestandstype vermelden.
4. Let op links die in een nieuw venster openen. Staat dat erbij?

### Voor developers

1. Open de Accessibility-tab in DevTools en inspecteer de toegankelijke naam van elke link. Beschrijft die het doel?
2. Gebruik een screenreader en open de linkslijst. In <span lang="en">VoiceOver</span>: <kbd>VO</kbd>+<kbd>U</kbd>. In NVDA: <kbd>Insert</kbd>+<kbd>F7</kbd>. Controleer of elke link op zichzelf begrijpelijk is.
3. Zoek in de codebase naar `>Klik hier<`, `>Lees meer<`, `>Meer<`, `>Hier<`.
4. Controleer alle links die alleen een afbeelding bevatten. Heeft de afbeelding een `alt`-attribuut dat het linkdoel beschrijft?
5. Zoek naar lege `<a>`-elementen of links met alleen een CSS-achtergrondafbeelding.
6. Controleer links met `target="_blank"`. Is er een indicatie dat de link in een nieuw venster opent?
7. Gebruik [axe DevTools](https://www.deque.com/axe/browser-extensions/) of [WAVE](https://wave.webaim.org/) om links zonder toegankelijke naam automatisch te vinden.
