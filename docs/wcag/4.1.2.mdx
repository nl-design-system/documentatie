## Korte samenvatting

Geef interactieve elementen zoals buttons, links en formuliervelden een toegankelijke naam, een bijpassende rol, toestand (state), eigenschappen en een waarde.

Waarom is dit nodig? Gebruikers van een screenreader krijgen deze informatie voorgelezen en weten zo hoe een interactief element te bedienen en wat de huidige status is.
Voor stembesturing is het belangrijk dat een element te activeren is, dit kan door de toegankelijke naam uit te spreken.

We geven een paar voorbeelden.

### Radiobutton
In een formulier kun je je favoriete kleur kiezen uit een paar kleuren.
Een geselecteerde radiobutton (keuzerondje) heeft de naam 'Blauw', de rol van 'radio' en de state is 'checked'.
De naam komt van het label, de rol komt door het inputelement type="radio" te geven en de waarde checked wordt door de browser gezet als de gebruiker `Blauw` selecteert.

In code:
```
<input id="kleur1" type="radio">
<label for="kleur1">Blauw</label>
```
De screenreader VoiceOver leest dan voor: 'Blauw, geselecteerd, keuzerondje'.
Deze informatie kun je ook terugvinden in de zogenaamde [accessibility tree](https://developer.chrome.com/blog/full-accessibility-tree).

Chrome laat voor dit voorbeeld zien voor het inputveld:
- Name: Blauw
- From label (for=attribute): label Blauw
- Role: radio
- Checked: true

### Link naar de contactpagina

De linktekst 'Contact' is de toegankelijke naam, de rol is link en de waarde is de url 'https://nldesignsystem.nl/project/kernteam'

 code:
```
<a href="https://nldesignsystem.nl/project/kernteam">Contact</a>
```
De screenreader VoiceOver leest dan voor: 'Link, contact'. En als de link al een keer aangeklikt is: 'Bezocht, Link, contact'


### Een button voor het openen en sluiten van een menu.

In dit voorbeeld heeft de button (knop) de toegankelijke naam 'Menu', de rol van button en de state expanded (open) of collapsed (gesloten).

Code voor het gesloten menu.
```
<button aria-expanded="false">Menu</button>
```
De screenreader VoiceOver leest dan voor: 'Menu, samengevouwen, knop'.
screenshot

Code voor het open menu
```
<button aria-expanded="true">Menu</button>
```
De screenreader VoiceOver leest dan voor: 'Menu, uitgevouwen, knop'.

De accessibility tree van Chrome vermeldt:
- Name: Menu
- Contents: "Menu"
- Role: button
- Expanded: true

**Let op**: Bij het gebruik van HTML is de rol al gedefinieerd voor de browser en hoef je geen rol toe te voegen. Gebruik je op maat gemaakte webcomponenten zonder gebruik te maken van HTML, dan zul je de naam, rol en waarde met ARIA en de toetsenbordinteractie met bijvoorbeeld JavaScript moeten toevoegen.
