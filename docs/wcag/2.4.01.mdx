<Summary />

Zo werken deze drie mechanismen:

- Een **skiplink** bovenaan de pagina die direct naar de hoofdinhoud springt.
- **Landmarkregio's** zoals `<main>`, `<nav>` en `<aside>` waarmee screenreadergebruikers per sectie door de pagina kunnen bewegen.
- **Koppen** waarmee screenreadergebruikers direct naar de juiste sectie springen.

Een skiplink werkt voor iedereen die met het toetsenbord werkt. Landmarkregio's en koppen helpen specifiek screenreadergebruikers.

## Veelgemaakte fouten

### Geen skiplink aanwezig

De pagina heeft geen skiplink. Een toetsenbordgebruiker gaat met <kbd>Tab</kbd> op elke pagina opnieuw door de volledige navigatie. Pas daarna komt de hoofdinhoud in beeld. Op een website met een uitgebreide header, een zoekbalk en een submenu kan dat tientallen tabstops kosten.

**Hoe te testen:** zet de cursor in de adresbalk van de browser en druk op <kbd>Tab</kbd>. Wat is het eerste focuspunt op de webpagina? Verschijnt er een link waarmee je direct naar de hoofdinhoud kunt springen?

<Guideline appearance="dont" title="Pagina zonder skiplink">
  ```html
  <body>
    <header>
      <nav>
        <a href="/">Home</a>
        <a href="/diensten">Diensten</a>
        <a href="/over-ons">Over ons</a>
        <a href="/contact">Contact</a>
      </nav>
    </header>
    <main>
      <h1>Welkom</h1>
    </main>
  </body>
  ```
</Guideline>

<Guideline appearance="do" title="Skiplink als eerste element op de pagina">
  ```html
  <body>
    <a href="#main" class="skiplink">
      Ga naar inhoud
    </a>
    <header>
      <nav>
        <a href="/">Home</a>
        <a href="/diensten">Diensten</a>
        <a href="/over-ons">Over ons</a>
        <a href="/contact">Contact</a>
      </nav>
    </header>
    <main id="main" tabindex="-1">
      <h1>Welkom</h1>
    </main>
  </body>
  ```
</Guideline>

Het `tabindex="-1"` op het `<main>`-element zorgt dat de focus daadwerkelijk naar het element verplaatst.

**Wie kan dit oplossen:** een developer voegt een skiplink toe aan het paginatemplate.

### Skiplink is er maar niet zichtbaar bij focus

De skiplink staat wel in de HTML, maar is permanent verborgen met `display: none` of `visibility: hidden`. Hierdoor kun je de link niet bereiken met het toetsenbord. Een skiplink mag visueel verborgen zijn totdat die focus krijgt. Zodra de link focus krijgt, wordt hij zichtbaar.

**Hoe te testen:** druk op <kbd>Tab</kbd> op een pagina. Verschijnt er een zichtbare skiplink?

<Guideline appearance="dont" title="Skiplink permanent verborgen">
    ```css
    .skiplink {
      display: none;
    }
    ```
</Guideline>

<Guideline appearance="do" title="Skiplink verborgen tot focus, dan zichtbaar">
    ```css
    .skiplink {
      position: absolute;
      left: -9999px;
    }

    .skiplink:focus {
      position: static;
      left: auto;
    }
    ```

</Guideline>

**Wie kan dit oplossen:** een developer past de CSS aan zodat de skiplink zichtbaar wordt bij focus.

### Skiplink verwijst naar een verkeerd of niet-bestaand doel

De `href` van de skiplink verwijst naar een `id` die niet op de pagina staat, of naar het verkeerde element. De link doet niets of springt naar een onverwachte plek.

**Hoe te testen:** activeer de skiplink met <kbd>Enter</kbd> en druk op <kbd>Tab</kbd>. Bevindt de focus zich in de hoofdcontent?

<Guideline appearance="dont" title="Skiplink verwijst naar een id dat niet bestaat">
    ```html
    <a href="#content">Ga naar inhoud</a>
    <!-- ... -->
    <main id="main">
      <h1>Welkom</h1>
    </main>
    ```
</Guideline>

<Guideline appearance="do" title="Skiplink en doel-id komen overeen">
    ```html
    <a href="#main">Ga naar inhoud</a>
    <!-- ... -->
    <main id="main" tabindex="-1">
      <h1>Welkom</h1>
    </main>
    ```
</Guideline>

**Wie kan dit oplossen:** een developer controleert of het `href`-attribuut van de skiplink overeenkomt met het `id` van het doelelement.

### Herhaald blok binnen de hoofdcontent heeft geen skip-mechanisme

Niet alleen de navigatie bovenaan de pagina is een herhaald blok. Een zijkolom met navigatie of een filterbalk die zich op meerdere pagina's herhaalt, valt ook onder "herhaalde content". Ook deze blokken moet je kunnen overslaan.

**Hoe te testen:** ga met <kbd>Tab</kbd> door de pagina. Zijn er filters of navigaties in de zijkolom die je bij elke pagina opnieuw moet doorlopen?

<Guideline appearance="do" title="Filterbalk in een gelabeld nav-element dat overgeslagen kan worden via landmarks">
    ```html
    <main id="main" tabindex="-1">
      <h1>Zoekresultaten</h1>
      <nav aria-label="Filters">
        <button>Categorie</button>
        <button>Prijs</button>
        <button>Beoordeling</button>
        <!-- ... nog meer filters -->
      </nav>
      <section aria-label="Resultaten">
        <h2>12 resultaten</h2>
        <!-- ... -->
      </section>
    </main>
    ```
</Guideline>

**Wie kan dit oplossen:** een developer plaatst herhaalde blokken binnen de hoofdinhoud in een eigen landmark met een duidelijke `aria-label`, zodat screenreadergebruikers het blok kunnen overslaan.

## Best practice

### Gebruik landmarkregio's voor de hoofdsecties van de pagina

Landmarkregio's zoals `<main>`, `<nav>`, `<header>` en `<footer>` geven screenreadergebruikers de mogelijkheid om per sectie door de pagina te bewegen. Het is een belangrijk extra mechanisme om herhaalde blokken over te slaan, naast een skiplink. Generieke `<div>`-elementen bieden die structuur niet.

**Hoe te testen:** open de pagina in een screenreader en open de landmarklijst. In <span lang="en">VoiceOver</span>: <kbd>VO</kbd>+<kbd>U</kbd> en kies "Landmarks". In <span lang="en">NVDA</span>: druk op <kbd>D</kbd> om naar de volgende landmark te springen. Zijn de belangrijkste secties als landmark herkenbaar?

<Guideline appearance="dont" title="Geen landmarkregio's, alles in div-elementen">
  ```html
  <div class="header">
    <div class="nav">...</div>
  </div>
  <div class="content">...</div>
  <div class="sidebar">...</div>
  <div class="footer">...</div>
  ```
</Guideline>

<Guideline appearance="do" title="Semantische landmarkregio's">
  ```html
  <header>
    <nav aria-label="Hoofdnavigatie">...</nav>
  </header>
  <main>...</main>
  <aside>...</aside>
  <footer>...</footer>
  ```
</Guideline>

**Wie kan dit toepassen:** een developer vervangt generieke `<div>`-elementen door semantische HTML-elementen.

### Geef elk navigatieblok een eigen beschrijving

Een pagina kan meerdere `<nav>`-elementen bevatten — bijvoorbeeld een hoofdnavigatie, een broodkruimelpad en een footernavigatie. Geef elk daarvan een `aria-label` dat de functie beschrijft. In de landmarklijst van een screenreader is dan duidelijk welke navigatie wat is.

**Hoe te testen:** open de landmarklijst in een screenreader. Zijn de navigatieregio's van elkaar te onderscheiden?

<Guideline appearance="dont" title="Twee nav-elementen zonder label">
  ```html
  <nav>
    <a href="/">Home</a>
    <a href="/diensten">Diensten</a>
  </nav>
  <nav>
    <a href="/">Home</a> &gt; <a href="/diensten">Diensten</a> &gt; Audit
  </nav>
  ```
</Guideline>

<Guideline appearance="do" title="Elke nav met een beschrijvend label">
  ```html
  <nav aria-label="Hoofdnavigatie">
    <a href="/">Home</a>
    <a href="/diensten">Diensten</a>
  </nav>
  <nav aria-label="Broodkruimelpad">
    <a href="/">Home</a> &gt; <a href="/diensten">Diensten</a> &gt; Audit
  </nav>
  ```
</Guideline>

**Wie kan dit toepassen:** een developer voegt een `aria-label` toe aan elk `<nav>`-element.

## Hoe te testen

### Voor iedereen

1. Laad een pagina en druk op <kbd>Tab</kbd>. Verschijnt er een skiplink? Activeer de link met <kbd>Enter</kbd>. Spring je voorbij de navigatie naar de hoofdcontent?
2. Bezoek een tweede pagina op dezelfde website en herhaal de test. De skiplink werkt op elke pagina.
3. Controleer of de skiplink zichtbaar wordt zodra die focus krijgt. Een skiplink die nooit zichtbaar is, is niet bruikbaar voor ziende toetsenbordgebruikers.
4. Controleer of er na de skiplink niet alsnog veel tabstops zijn voordat je bij de content bent. Dat kan wijzen op een verkeerd doel of een extra herhaald blok zonder skip-mechanisme.

### Voor developers

1. Controleer of de pagina een skiplink bevat als eerste focusbare element. De `href` verwijst naar een `id` op het `<main>`-element of het begin van de hoofdcontent.
2. Controleer of het doelelement van de skiplink `tabindex="-1"` heeft, zodat de focus daadwerkelijk verplaatst. Zonder `tabindex` springt de focus in sommige browsers terug.
3. Controleer of de skiplink niet permanent verborgen is met `display: none` of `visibility: hidden`. De link is visueel verborgen tot focus en dan zichtbaar.
4. Inspecteer de landmarkregio's op de pagina. Zijn `<header>`, `<nav>`, `<main>`, `<aside>` en `<footer>` aanwezig? Hebben meerdere `<nav>`-elementen elk een eigen `aria-label`?
5. Controleer of de pagina een `<h1>` heeft aan het begin van de hoofdcontent. Screenreadergebruikers navigeren via koppen als alternatief voor een skiplink.
6. Gebruik <span lang="en">axe DevTools</span> of <span lang="en">WAVE</span> voor een eerste scan. Deze tools herkennen een ontbrekende landmarkregio of een missende skiplink niet altijd. Of de skiplink correct werkt, controleer je handmatig.

## Gerelateerde succescriteria

- [2.4.3 Focusvolgorde](/wcag/2.4.3/): de skiplink is onderdeel van de focusvolgorde. Als de focusvolgorde niet klopt, kan de skiplink op een onverwachte plek staan.
- [2.4.6 Koppen en labels](/wcag/2.4.6/): beschrijvende koppen zijn een tweede mechanisme om blokken te omzeilen. Screenreadergebruikers springen via de koppenlijst direct naar de juiste sectie.
- [1.3.1 Info en relaties](/wcag/1.3.1/): landmarkregio's en koppen zijn programmatisch bepaald. Zonder correcte HTML-structuur werken deze mechanismen niet.
- [2.4.2 Paginatitel](/wcag/2.4.2/): de paginatitel is het eerste wat een screenreader voorleest. Samen met de skiplink en landmarks helpt het bij snelle oriëntatie.
