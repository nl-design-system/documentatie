De Candidate component kan makkelijk worden gebruikt in diverse projecten zodat er genoeg feedback wordt gegeven door developers om er uiteindelijk een Hall of Fame component van te maken.

Developers kunnen Candidate componenten herkennen aan `@nl-design-system-candidate/` in de naam.

Developers kiezen voor de Candidate component wanneer ze akkoord zijn met frequente releases, waar duidelijke migratiestappen zijn voor breaking changes.

Developers gebruiken de Candidate component wanneer ze een component willen gebruiken die zoveel mogelijk voldoet aan de Definition of Done, terwijl er nog geen component is die 100% voldoet.
