# Werken met pnpm

pnpm is een snelle, efficiÃ«nte package manager voor Node.js.
Het is een alternatief voor npm en yarn en staat bekend om zijn zuinige gebruik van schijfruimte en hoge performance.
Daarnaast biedt pnpm een aantal beveiligingsfeatures waar we gebruik van maken.

Ben je npm gewend?
De meeste commando's zijn hetzelfde: je gebruikt bijvoorbeeld `pnpm install` in plaats van `npm install`.
Bekijk de documentatie op de website van [pnpm](https://pnpm.io/pnpm-cli) voor uitgebreide informatie.

## De juiste pnpm-versie installeren en gebruiken

Tegenwoordig hoef je de package manager niet meer los te installeren, maar gebruik je [Corepack](https://github.com/nodejs/corepack).
Dit is in recente versies van [Node.js](/developer/node) inbegrepen.
Corepack gebruikt automatisch de versie van pnpm die in `package.json` staat, zodat teams en CI dezelfde versie gebruiken.

```sh
# activeer Corepack
corepack enable

# controleer de installatie
pnpm -v
```

Je vindt de pnpm-minimumversie in `package.json` onder `engines.pnpm`. De minimumversie is nu {getMinimumPnpmVersion()}. Deze versie beschermt je beter via de [minimumleeftijd van een release](https://pnpm.io/settings#minimumreleaseage).

## pnpm upgraden

Gebruik ook voor het upgraden Corepack. Upgrade de versie in `package.json` altijd via de `corepack` CLI. Zo voegt Corepack een hash toe aan de versie. Dit is een extra beveiligingsmaatregel, waarmee je controleert of elke machine exact dezelfde pnpm-binary gebruikt.

```sh
# upgraden naar laatste versie in major range
corepack up

# of: upgraden naar specifieke versie
corepack use pnpm@10
```
