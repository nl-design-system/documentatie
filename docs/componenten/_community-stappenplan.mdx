## Implementatie

## Minimale design tokens

Doel: Meerdere organisaties kunnen het component naar hun huisstijl omzetten.

Door met [design tokens](/handboek/design-tokens/) te werken zorgen we ervoor dat meerdere organisaties het component van een eigen huisstijl kunnen voorzien. Minimaal zouden er design tokens beschikbaar moeten zijn om kleur en typografie beslissingen door te voeren.

Zorg ervoor dat het component van jouw organisatie gebruik maakt van design tokens. Of controleer de implementaties van organisaties die aan de GitHub Discussion van het component zijn toegevoegd.

**Tip!** Om te controleren of er design tokens zijn toegepast kun je gebruik maken van de 'Inspect' functionaliteit van je browser. Wanneer je in de CSS verwijzingen ziet naar CSS variabelen zoals bijvoorbeeld `--denhaag-button-primary-action-background-color` of `--ams-button-font-family` kun je er vanuit gaan dat er design tokens zijn gebruikt.

**üö© Checkpoint**: Implementatie heeft minimaal kleur en typografie beslissingen met design tokens gebruikt.

Zorg ervoor dat jouw organisatie naar de EUPL-1.2 licentie verwijst op de onderstaande posities. Of controleer de organisaties die aan de GitHub Discussion van het component zijn toegevoegd.

- **GitHub repository**: Als √©√©n van de repository details onder 'About' in de sidebar.

**üö© Checkpoint**: EUPL-1.2 licentie toegepast.

## Licentie documentatie

Doel: De documentatie van een component mag door andere organisaties hergebruikt of gecombineerd worden zonder enige bronvermelding.

Binnen het NL Design System werken we voor de documentatie met de [Creative Commons 0 licentie (CC0)](https://nldesignsystem.nl/open-source/cc0).

Zorg ervoor dat jouw organisatie de CC0 licentie benoemt in het README.md bestand van het component in GitHub. Of controleer de organisaties die aan de GitHub Discussion van het component zijn toegevoegd.

**Tip!** Door de 'Code' of 'Raw' weergave van het README.md bestand te bekijken zou je bovenaan `<!-- @license CC0-1.0 -->` moeten zien staan.

**üö© Checkpoint**: Documentatie heeft de Creative Commons 0 licentie (CC0).

## Prefix

Om componenten van elkaar te kunnen gebruiken is het toevoegen van een prefix zoals bijvoorbeeld `utrecht-` of `ams-` verplicht.

Zorg ervoor dat de prefix van jouw organisatie is toegevoegd aan onderstaande onderdelen. Of controleer de organisaties die aan de GitHub Discussion van het component zijn toegevoegd.

- CSS class selectors.
- Design tokens in JSON.
- CSS variables.

**üö© Checkpoint**: API's hebben een prefix van de verantwoordelijke organisatie.
